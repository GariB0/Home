1. About HTML 5

With HTML5 for common tasks, developers can rely on built-in browser capabilities and not need to reinvent the wheel for every application.

1) XHTML History
The first version, XHTML 1.0, is nearly identical to HTML 4.01, sharing the same elements and attributes, but with stricter requirements for how markup must be done.
HTML 4.01, along with XHTML 1.0, its stricter XML-based sibling, became the cornerstone of the web standards movement...
In 2004, members of Apple, Mozilla, and Opera formed the Web Hypertext Application Technology Working Group separate from the W3C. The goal of the WHATWG was to further the development of HTML to meet new demands in a way that was consistent with real-world authoring practices and browser behavior.
The W3C eventually established its own HTML5 Working Group based on the work done by the WHATWG.
As of this writing, work on the HTML5 specification is happening in both organizations in tandem, sometimes with conflicting results.
Their initial documents, Web Applications 1.0 and Web Forms 1.0, were rolled together into HTML5, which is still in development under the guidance of an editor, Ian Hickson (currently of Google).

2) New features that HTML5 offers
A new DOCTYPE
New elements and attributes
Obsolete 4.01 elements
APIs
 A minimal DOCTYPE(HTML5): <! DOCTYPE html >

2. How Style Sheets Work

A style sheet is made up of one or more style instructions that describe how an element or group of elements should be displayed.
h1 { color: green; } // selects h1; identifies the element or elements to be affected.
p { font-size: small; ... } // declares font-size; provides the rendering instructions.
* the declaration is made up of a property and its value.

Attaching the styles to the document; External & Embedded style sheets.
External style sheets: <script src="file.css" > </script>
Embedded style sheets: <script > /* style rules go here */</script>
* Inline styles: <element style="the rules go here" > ~ </element>

3. Inheritance

Styled HTML elements pass down certain style properties to the elements they contain.

1) Document structure
HTML documents have an implicit structure or hierarchy.
example: HTML - head(left) & body(right)
Html root element that contains a head and a body, and the body contains heading and paragraph elements.
* The other elements have the structure as an upside-down tree, branching out from the root.

2) Parents and children
The document tree becomes a family tree when it comes to referring to the relationship between elements.
All the elements contained within a given element are said to be its descendants.
example: the h1, h2, p, em, and img element are all descendants of the body element.
Conversely, the containing element is the parent.
All of the elements higher than a particular element in the hierarchy are its ancestors. Two elements with the same parent are siblings.

3) Pass it on
When you write a font-related style rule using the p element as a selector,
the rule applies to all of the paragraphs in the document as well as the inline text elements they contain.
* In general, properties related to the styling of text font size, color, style, etc., are passed down.
Properties such as borders, margins, backgrounds, and so on, that affect the boxed area around the element tend not to be passed down.

4. Conflicting styles

CSS allows you to apply several style sheets to the same document, which means there are bound to be conflicts.
The cascade refers to what happens when several sources of style information vie for control of the elements on a page:
style information is passed down (“cascades” down) until it is overridden by a style command with more weight.

1) Weight & importance
Generally speaking, the closer the style sheet is to the content, the more weight it is given.
Embedded style sheets that appear right in the document in the style element have more weight than external style sheets.
To prevent a specific rule from being overridden, you can assign it“importance” with the !important indicator

2) Specificity
Once the applicable style sheet has been chosen, there may still be conflicts;
When two rules in a single style sheet conflict, the type of selector is used to determine the winner.
The more specific the selector, the more weight it is given to override conflicting declarations.

3) Rule order
if there are conflicts within style rules of identical weight, whichever one comes last in the list “wins.”

5. The box model & Grouped selectors

1) The box model
The easiest way to think of the box model is that browsers see every element on the page (both block and inline) as being contained in a little rectangular box. You can apply properties such as borders, margins, padding, and backgrounds to these boxes, and even reposition them on the page.

2) Grouped selectors
A handy style rule being shortcut: If you ever need to apply the same style property to a number of elements,
you can group the selectors into one rule by separating them with commas.
example: h1, h2, p, em, img { border: 1px solid blue; }
