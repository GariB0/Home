Learning Order

1. Character Set

JavaScript programs are written using the Unicode character set. Unicode is a superset of ASCII and Latin-1 and supports virtually every written language currently used on the planet.

1) Case Sensitivity language
Language keywords, variables, function names, and other identifiers in Javascripts must always be typed with a consistent capitalization of letters.
 * Note that HTML is not case-sensitive (although XHTML is).

2) Control Characters(Whitespace, Line Breaks...)
JavaScript ignores line breaks because you can use spaces and newlines freely in your programs, you can format and indent your programs in a neat and consistent way that makes the code easy to read and understand.

3) Escape Sequences
Some computer hardware and software can not display or input the full set of Unicode characters.
To support programmers using this older technology, JavaScript defines special sequences of six ASCII characters to represent any 16-bit Unicode codepoint.
 * Unicode

4) Normalization
Unicode allows more than one way of encoding the same character.
The Unicode standard defines the preferred encoding for all characters and specifies a normalization procedure to convert text to a canonical form suitable for comparisons. JavaScript assumes that the source code it is interpreting has already been normalized and makes no attempt to normalize identifiers, strings, or regular expressions itself.

2. Literals

A literal is a data value that appears directly in a program.

1) numbers
2) string
3) A boolean value
4) A regular expression literal for pattern matching
5) Objects
6) Coment

3. Identifiers

Identifiers are used to name variables and functions and to provide labels for certain loops in JavaScript code.
For portability and ease of editing, it is common to use only ASCII letters and digits in identifiers.
JavaScript reserves certain identifiers for use by the language itself which is called reserved words.

For example, The words, such as 'break','do',etc, cannot be used as identifiers.

4. Optional Semicolons

Client-side JavaScript does not exhibit the nonlinear cross-reference problem nearly to the extent that the core language does, and it is possible to learn how to use JavaScript in web browsers in a fairly linear sequence.

* Basic Concepts

The lexical structure of a programming language is the set of elementary rules that specifies how you write programs in that language.

1) Syntax of an language
It is the rules that defines the combinations of symbols that are considered to be correctly structured statements or expressions in that language.

2) Unicode character set
Unicode, formally The Unicode Standard, is an information technology standard for the consistent encoding, representation, and handling of text expressed in most of the world's writing systems.
* The standard defines as of the current version (15.0) 149,186 characters covering 161 modern and historic scripts, as well as symbols, etc.

3) Identifiers
An identifier is a name that identifies (that is, labels the identity of) either a unique object or a unique class of objects, where the "object" or class may be an idea, physical countable object (or class thereof), or physical noncountable substance (or class thereof).
