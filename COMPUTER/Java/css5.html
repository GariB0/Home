Learning Order

1. Normal Flow & float

Text elements are laid out from top to bottom in the order in which they appear in the source, and from left to right.
Block elements stack up on top of one another and fill the available width of the browser window or other containing element.
Inline elements and text characters line up next to one another to fill the block elements.

Objects in the normal flow affect the layout of the objects around them. This is the behavior you’ve come to expect in web pages; elements don't overlap or bunch up.

1) The float property
The float property moves an element as far as possible to the left or right,
allowing the following content to wrap around it.
float Property:

Values: left|right|none|inherit
Default: none
Applies to: all elements
Inherits: no

2) example: default
When Inline image in the normal flow(float:default) space next to the image is held clear.

3) example: right floating
When Inline image floated to the right space, image moves over and text wraps around it.

2. Floating inline & block elements

With CSS, it is possible to float any HTML element, both inline and block-level.

1) Floating an inline text element
Always provide a width for floated text elements.
Floated inline elements behave as block elements.
Margins on floated elements do not collapse.

2) Floating block elements
You must provide a width for floated block elements;
If you do not provide a width value, the width of the floated block will be set to auto.
Elements do not float higher than their reference in the source.

3) Clearing floated elements
Clear is to turn the block-level elemnts wrapping off and get back to layout as usual. clear Property:
Values: left|right|both|none|inherit
Default: none
Applies to: block-level elements only
Inherits: no

3. Multiple & Containing floats

floating multiple elements: there is a complex system of behind-thescenes rendering rules that ensures floated elements do not overlap.
containing floats: another float quirk.

1) Floating multiple elements
When a series of sequential paragraphs are floated to the same side:
The first three floats start stacking up from the left edge, but when there isn’t enough room for the fourth, it moves down and to the left until it bumps into something
in this case, the edge of the browser window.
* The other way to make list items line up is to make them display as inline elements instead of as block elements:{display: inline;}.

2) Containing floats
By default, floats are designed to hang out of the element they are contained in.
The border stretched to contain all the content, however, the floated image hangs right out the bottom.
And if you float all the elements in a container element there will be no elements remaining in the flow to hold the containing element open.
* The other common solution is the overflow property instead of the float;
Setting the overflow of the containing element to auto or hidden will also make it stretch to contain the floated elements.

3) Using floats to create columns
For a two-column float, you can do the following:
a. Float one div and add a wide margin on the side of the text element that wraps around it.
b. Float both divs to the left or right.
c. Float one div to the left and the second div to the right (or vice versa).
* Three-column floats work basically the same way; there's just more calculating to do.

4. Positioning Concept

CSS provides several methods for positioning elements on the page.

1) Types of positioning
The position property indicates that an element is to be positioned and specifies which positioning method to use.
Static value is the normal positioning scheme in which elements are positioned as they occur in the normal document flow.
Relative positioning moves the box relative to its original position in the flow.
Absolutely positioned elements are removed from the document flow entirely and positioned with respect to the browser window or a containing element.
The distinguishing characteristic of fixed positioning is that the element stays in one position in the window even when the document scrolls.

position Property:
Values: static|relative|absolute|fixed|inherit
Default: static
Applies to: allelements
Inherits: no

2) Relative Positioning
The values of the offset properties defines the distance the element should be moved away from that respective edge.

offset properties(top,bottom,right,left):
Values: length measurement|percentage|auto|inherit
Default: auto
Applies to: positioned elements(relative,absolute,or fixed)
Inherits: no

Relative positioning moves an element relative to its original spot in the flow
The space it would have occupied is preserved and continues to influence the layout of surrounding content.
example:
em {
position: relative;
top: 30px;
left: 60px;
}
* Overlap happens.

3) Absolute Positioning
It is actually a more flexible method for accurately placing items on the page than relative positioning.

example:
em {
position: absolute;
top: 30px;
left: 60px;
}
* In its new position, the element box overlaps the surrounding content. In the end, absolutely positioned elements have no influence whatsoever on the layout of surrounding elements. The most significant difference here, however, is the location of the positioned element. This time, the offset values position the em element 30 pixels down and 60 pixels to the right of the top-left corner of the browser window.
* Note that that the element is positioned relative to its nearest containing block.

4) Containing blocks
The position and size of an element's box(es) are sometimes calculated relative to a certain rectangle.
The recommendation lays out a number of intricate rules for determining the containing block of an element:

If the positioned element is not contained within another positioned element, then it will be placed relative to the initial containing block
But if the element has an ancestor that has its position set to relative, absolute, or fixed, the element will be positioned relative to the edges of that element instead.

5. Specifying position

It is to get better acquainted with the offset properties.

1) Pixel measurements
The positive offset values push the positioned element box away from the specified edge and toward the center of the containing block.
If there is no value provided for a side, it is set to auto, and the browser adds enough space to make the layout work.

example:
div#a {
position: relative; /* creates the containing block */
height: 120px;
width: 300px;
border: 1px solid;
background-color: #CCC;
}
div#b {
position: absolute;
top: 20px;
right: 30px;
bottom: 40px;
left: 50px;
border: 1px solid;
background-color: teal;
}

* Notice that by setting offsets on all four sides, I have indirectly set the dimensions of the positioned div#b (it fills the 220 × 60 pixel space that is left over within the containing block after the offset values are applied). If I had also specified a width and other box properties for div#b, there is the potential for conflicts if the total of the values for the positioned box and its offsets do not match the available space within the containing block.

2) Percentage values
You can also specify positions with percentage values.

example:
img#A {
position: absolute;
top: 50%;
left: 0%; /* the % symbol could be omitted for a 0 value */
}
img#B {
position: absolute;
bottom: 0%; /* the % symbol could be omitted for a 0 value */
right: 0%; /* the % symbol could be omitted for a 0 value */
}

* Although the examples here specify both a vertical and horizontal offset, it is common to provide just one offset for a positioned element, for example, to move it left or right into a margin using either left or right properties.

6. Stacking Order & Fixed position

1) Stacking Order
Absolutely positioned elements overlap other elements, so it follows that multiple positioned elements have the potential to stack up on one another. By default, elements stack up in the order in which they appear in the document, but you can change the stacking order with the z-index property.

z-index Property:
Values: number|auto|inherit
Default: auto
Applies to: positioned elements(relative,absolute,or fixed)
Inherits: no
* The value of the z-index property is a number (positive or negative).

2) Fixed position
For the most part, fixed positioning works just like absolute positioning. The significant difference is that the offset values for fixed elements are always relative to the viewport, which means the positioned element stays put even when the rest of the page scrolls.

example:
#award {
position: fixed;
top: 35px;
left: 25px;
}
